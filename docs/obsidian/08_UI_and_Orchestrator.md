## Как UI должен вызывать оркестратор

Гипотеза: UI передает оркестратору текст задачи, контекст пользователя, выбранный сценарий и ограничения. Оркестратор решает, нужны ли память, 1С, шаблоны КП или LLM.

## Где может быть бизнес-логика

Бизнес-логика КП и правил сохранения опыта должна жить не в UI и не в низкоуровневом `MemoryStore`, а в отдельном application/service слое.

## Что не смешивать

- UI не должен напрямую писать неподтвержденный опыт в episodic memory.
- MemoryStore не должен делать вызовы 1С.
- Оркестратор не должен хранить credentials.
- LLM не должен получать больше данных, чем нужно для ответа.
