## Архитектура

### TD-001 Нет слоя загрузки документов

| Поле | Значение |
| --- | --- |
| Категория | архитектура |
| Описание | В проекте нет модуля, который читает PDF/DOCX/TXT, режет документы на chunk и передает их в `MemoryStore.add_chunk()`. |
| Где проявляется | Отсутствуют loader/chunker модули; см. [[09_Document_Loading]]. |
| Почему проблема | Semantic memory нельзя наполнять документами автоматически. |
| Риск | высокий |
| Приоритет | высокий |
| Сложность | средняя |
| Предлагаемое решение | Создать слой загрузки документов, определить metadata, reindex и тесты. |
| Связанный roadmap | [[15_Roadmap#До MVP]] |
| Связанные решения/вопросы | [[16_Decision_Log#ADR-001 SQLite как MVP-хранилище памяти]], [[18_Open_Questions#Данные]] |

### TD-003 Нет оркестратора и границ бизнес-логики

| Поле | Значение |

| Связанный roadmap | [[15_Roadmap#До MVP]] |

## Память

### TD-005 Нет политики безопасного сохранения памяти

| Поле | Значение |
| --- | --- |
| Категория | память, безопасность |
| Описание | `add_chunk()` и `save_experience()` не проверяют чувствительные данные, подтверждение пользователя и допустимость metadata. |

| Почему проблема | Можно случайно сохранить неподтвержденный опыт, данные клиента или оперативные данные из 1С. |
| Риск | высокий |
| Приоритет | высокий |
| Сложность | средняя |
| Предлагаемое решение | Добавить data policy, слой подтверждения, санитайзер и тесты. |
| Связанный roadmap | [[15_Roadmap#Ближайшие задачи]] |
| Связанные решения/вопросы | [[16_Decision_Log#ADR-002 Разделение semantic и episodic memory]], [[18_Open_Questions#Безопасность]] |

### TD-002 Поиск только FTS5 без embeddings

| Поле | Значение |
| --- | --- |
| Категория | поиск |
| Описание | Реализован keyword/full-text search через FTS5, но нет embeddings, hybrid merge и rerank. |
| Где проявляется | `MemoryStore.search_fts()` в [memory_store.py](../../linehelper/memory/memory_store.py). |
| Почему проблема | Качество поиска зависит от совпадения слов и формулировок. |
| Риск | средний |
| Приоритет | высокий |
| Сложность | средняя |
| Предлагаемое решение | Добавить embeddings после стабилизации loader, затем hybrid search и rerank. |
| Связанный roadmap | [[15_Roadmap#После MVP]] |
| Связанные решения/вопросы | [[16_Decision_Log#ADR-005 Возможная миграция на vector DB]], [[07_RAG_Search]] |
