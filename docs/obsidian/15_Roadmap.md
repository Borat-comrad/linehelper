# Roadmap

## Ближайшие задачи

1. Зафиксировать правила, что можно сохранять в semantic и episodic memory.
2. Добавить `.env.example` без секретов.
3. Спроектировать loader/chunker для TXT/PDF/DOCX.
4. Добавить тесты на русскоязычный поиск и metadata.
5. Описать контракт будущего оркестратора.

## До MVP

- Загрузка документов в semantic memory.
- Reindex документов.
- Source citation.
- Минимальный оркестратор.
- Минимальный UI или CLI.
- Интеграция с тестовым контуром 1С.
- КП-flow с подтверждением пользователя.
- End-to-end smoke-test.

## После MVP

- Hybrid search.
- Embeddings.
- Rerank.
- Qdrant или другая vector DB.
- Роли и права доступа.
- Audit log.
- Улучшение качества ответов на реальных сценариях.

## Эволюция SQLite -> vector DB

Фаза 1: SQLite + FTS5 как единственный storage.

Фаза 2: SQLite хранит текст и metadata, отдельный embedding pipeline считает векторы.

Фаза 3: vector DB хранит embeddings, SQLite остается источником metadata и трассировки.

Фаза 4: hybrid retriever объединяет FTS, vector и бизнес-фильтры.

## Критерии готовности MVP

- Пользователь может задать вопрос и получить ответ с источниками.
- Документ можно загрузить, переиндексировать и найти.
- 1С-данные используются как актуальный источник, а не как память.
- КП можно подготовить в черновике и сохранить успешный опыт только после подтверждения.
- Тесты покрывают MemoryStore, загрузку, поиск и основной сценарий.
- Секреты и реальные рабочие данные не попадают в Git.
