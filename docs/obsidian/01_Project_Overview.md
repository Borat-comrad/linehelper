# Обзор проекта

LineHelper задуман как локальный AI-агент для Serviceline. Текущий код реализует фундамент этого агента: локальное долговременное хранилище памяти, которое в будущем сможет использоваться оркестратором.

## Зачем существует проект

Факт из README: проект должен хранить и находить корпоративные знания и подтвержденный практический опыт. Гипотеза по названию и README: конечная цель - помогать сотрудникам быстрее отвечать на рабочие вопросы и готовить коммерческие предложения с учетом документов, прошлого опыта и актуальных данных из 1С.

## Бизнес-проблема

Проект пытается разделить два типа знаний:

- стабильные корпоративные знания: регламенты, инструкции, оргструктура, ЦКП, ЗРС, документооборот;
- подтвержденный практический опыт: КП, сделки, предпочтения клиентов, рабочие аргументы.

Без такого разделения AI-агент может смешивать справочные знания, устаревший опыт и оперативные данные. См. [[05_Memory_System]].

## Основные пользователи

- Менеджер или специалист Serviceline, который ищет знания и готовит КП.
- Разработчик, который развивает локального агента.
- Руководитель, которому нужна карта состояния продукта и рисков.

Уточнить: точные роли пользователей, права доступа и сценарии работы в репозитории пока не описаны.

## Что входит в текущий MVP

- SQLite-база `data/memory/linehelper_memory.db`.
- Таблица `memory_chunks`.
- FTS5-таблица `memory_chunks_fts`.
- Namespace `semantic` и `episodic`.
- Добавление текстового chunk.
- Полнотекстовый поиск через FTS5.
- Сохранение подтвержденного опыта в episodic memory.
- Удаление chunk.
- Очистка устаревших episodic-записей.
- Pytest-тесты для MemoryStore.

## Что не входит в текущий MVP

По README явно не реализованы:

- загрузчик PDF/DOCX/TXT;
- автоматический chunking документов;
- embeddings;
- hybrid search;
- LLM;
- интеграция с 1С;
- UI.

## Границы системы

Текущая система - это Python-библиотека и скрипты вокруг локальной SQLite-БД. Это не веб-приложение и не полноценный агент.

Операционные данные должны приходить из внешних источников, например из 1С, и не должны автоматически становиться памятью. См. [[06_1C_Integration]].

## Операционные данные и память

| Тип данных | Где должны жить | Статус |
| --- | --- | --- |
| Регламенты, инструкции, справки | `semantic` memory | реализован storage, загрузка TODO |
| Подтвержденный опыт КП/сделок | `episodic` memory | реализован `save_experience()` |
| Актуальные цены, остатки, статусы заказов | 1С или другой оперативный источник | TODO |
| Черновики, неподтвержденные гипотезы | не сохранять как опыт без подтверждения | правило нужно закрепить |
